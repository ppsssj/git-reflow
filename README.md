# git-reflow

GitHub의 기본 화면을 사용자 설정에 따라 재구성하는 브라우저 확장 프로그램입니다.  
사용자는 피드 섹션의 노출 여부, 배치 순서, 강조 방식 등을 직접 조정할 수 있으며, 이를 통해 GitHub를 더 탐색 친화적인 UI로 사용할 수 있습니다.

## Overview

GitHub는 많은 정보를 제공하지만, 사용자가 원하는 방식으로 화면 구조를 바꾸기는 어렵습니다.  
특히 홈 피드나 추천 영역에서는 사용 목적과 관계없이 고정된 순서와 레이아웃이 제공되기 때문에, 원하는 정보를 빠르게 확인하기 어려운 경우가 있습니다.

git-reflow는 이러한 문제를 해결하기 위해 만들어진 확장 프로그램입니다.  
기존 GitHub UI 위에 사용자 맞춤형 프론트엔드 레이어를 적용하여, 화면의 정보 구조와 레이아웃을 재구성하는 것을 목표로 합니다.

이 프로젝트는 단순한 스타일 변경이 아니라, 다음과 같은 방향을 지향합니다.

- GitHub 홈 피드의 정보 우선순위 재조정
- 추천 섹션과 탐색 섹션의 노출 방식 개선
- 사용자 설정 기반 레이아웃 커스터마이징
- 탐색성과 정보 가시성 향상

- GitHub 홈 피드 섹션 표시/숨김
- 섹션 순서 재배치
- 추천 섹션 강조 표시
- 사용자 설정 저장 및 재적용
- 확장 프로그램 기반 UI 커스터마이징
- 기존 GitHub 화면 위에 오버레이 형태로 동작

## How It Works

git-reflow는 GitHub 서버를 수정하지 않습니다.  
대신 브라우저 확장 프로그램의 content script를 통해 현재 렌더링된 GitHub 페이지의 DOM과 CSS를 제어하여 화면 구조를 변경합니다.

기본 동작 방식은 다음과 같습니다.

1. 사용자가 GitHub 페이지에 접속합니다.
2. 확장 프로그램이 현재 페이지의 구조를 분석합니다.
3. 저장된 사용자 설정을 불러옵니다.
4. 섹션을 숨기거나, 강조하거나, 다른 위치로 재배치합니다.
5. 사용자는 자신에게 맞는 GitHub 레이아웃으로 페이지를 사용할 수 있습니다.

## Project Goals

이 프로젝트의 핵심 목표는 다음과 같습니다.

- GitHub의 고정된 UI 구조를 사용자 중심으로 재해석하기
- 정보 탐색 흐름을 더 빠르고 직관적으로 만들기
- 단순한 꾸미기 수준이 아니라, 실제 사용 목적에 맞는 레이아웃 제어 기능 제공하기
- 향후 GitHub UI 개인화 도구로 확장 가능한 구조 설계하기

## Use Cases

### 1. 추천 레포지토리를 더 자주 보고 싶은 사용자
추천 영역을 상단으로 올리고, 더 눈에 띄게 강조할 수 있습니다.

### 2. 불필요한 피드 블록을 줄이고 싶은 사용자
자주 보지 않는 섹션을 숨기고, 필요한 정보만 남길 수 있습니다.

### 3. GitHub를 탐색형 대시보드처럼 쓰고 싶은 사용자
홈 화면을 자신의 작업 방식에 맞는 정보 구조로 재구성할 수 있습니다.

## Why This Project

많은 웹서비스는 사용자에게 동일한 정보 구조를 제공합니다.  
하지만 실제 사용 맥락은 사용자마다 다릅니다. GitHub 역시 탐색, 협업, 오픈소스 발견, 개인 활동 확인 등 사용 목적이 다양합니다.

git-reflow는 GitHub의 기본 UI를 그대로 받아들이는 대신,  
사용자가 원하는 방식으로 화면을 조정할 수 있도록 만드는 데 초점을 둡니다.

즉, 이 프로젝트는 기존 서비스 위에 사용자 맞춤형 인터페이스 레이어를 구축하는 실험이자,  
브라우저 확장 프로그램을 활용한 프론트엔드 재구성 사례입니다.
